# Project Structure Overview

This Go project follows Domain-Driven Design (DDD) principles, with a clear separation of concerns across layers. The repository interfaces are minimal (list, count, get, save/upsert), and errors are mapped appropriately between infrastructure, domain, service, and delivery layers.

## Technologies

- **Languages & Frameworks:** Go, Gin, Google Wire, Swaggo
- **Databases & Tools:** PostgreSQL, sqlc
- **External Services:** S3 (MinIO), Keycloak, Redis

## Directory Layout

- **cmd/**
  Application entrypoint (main.go).

- **internal/**
  - **application/**
    Business logic, use cases, and parameter objects.
  - **client/**
    External adapters: database, Gin, Keycloak, Redis, S3, validation.
  - **di/**
    Dependency injection setup (Google Wire).
  - **domain/**
    Domain models, repository/service interfaces, domain errors, enums.
    Includes `*_repository_mock.go` files generated by mockery for testing (using testify).
  - **service/**
    Domain service implementations.

- **database/**
  - **schema.sql**
    Main database schema.
  - **queries/**
    sqlc query files.
  - **temporary-table/**
    Temporary table definitions for sqlc (not part of main schema).
  - Other SQL scripts (init, seed, triggers, etc.).

- **migration/**
  Database migration scripts.

- **build/**
  Dockerfiles for backend and migrations.

- **docker/**
  Docker Compose and volume initialization scripts.

- **terraform/**
  Infrastructure as code (Keycloak, MinIO).

- **keycloak/**
  Realm and client export files.

- **pkg/logger/**
  Logging utilities.

- **docs/**
  API documentation, Swagger files, and static HTML.

- **.rules/**
  Project rules and structure documentation.

- **.github/**
  CI/CD workflows.

- **.vscode/**
  Editor configuration.

- **.vectorcode/**
  VectorCode configuration.

- **Config files:**
  `.env`, `.gitignore`, `.editorconfig`, `.golangci.yaml`, etc.

## Notes

- All repository mocks (`*_repository_mock.go`) in `internal/domain` are auto-generated for unit testing.
- Delivery/controller layer is implemented as `internal/delivery` (not present in current scan, add if needed).
- Error mapping flows from infrastructure → domain → service → application → HTTP delivery.
